Sistema de Gestion de Alertas

Expanda las secciones para mayor información sobre el sistema.

Notificaciones

Este módulo permite el manejo de las notificaciones generadas por el sistema, litando aquellas cuyo estado sea "creada" o "en_proceso".

Dependiendo del tipo de usuario, se podrá acceder a las distintas funcionalidades de este módulo: Asignar, Resolver y Cerrar

Asignar: a través de esta función, los técnicos podrán asignarse las notificaciones. Una vez asignada, cambia su estado a "en_proceso".

Resolver: se podrán resolver unicamente aquellas notificaciones en estado "en_proceso" agregando un comentario sobre la resolucion aplicada.

Cerrar: luego de ingresar un comentario se podran cerrar las notificaciones en estado "en_proceso".

Reportes

En este módulo se pueden consultar las notificaciones del sistema aplicando alguno de los siguientes filtros:

- Fecha: lista las notificaciones comprendidas en el rango de fechas ingresado.

- Técnico: lista las notificaciones asignadas al tecnico seleccionado.

- Estado: lista las notificaciones con el estado seleccionado ("creada", "en_proceso", "cerrada", "cancelada") .

Nota: la cantidad de registros que retorna el reporte es configurable. Ver Manual de configuarción por mayor información.

Auditoria

Las acciones realizadas por los usuarios del sistema son almacenadas y pueden ser consultadas a través de éste médulo utilzando los siguientes filtros:

- Fecha: lista los registros comprendidos en el rango de fechas ingresado.

- Técnico: lista las acciones realizadas por el técnico seleccionado.

- Acción: lista los registros que contienen la acción ingresada.

- Tipo de Acción: lista los registros cuya accion coincide con la ingresada ("info", "error").

Nota: la cantidad de registros que retorna el reporte de auditoria es configurable. Ver Manual de configuarción por mayor información.

Usuarios

Este módulo se utiliza para administrar los usuarios del sistema.

Crear Usuario: se deberá completar los datos de Usuario, Mail, Tipo de Usuario y Contraseña; La informacion de Nombre y Mail no es obligatoria.

Editar: permite modificar la informacion del Nombre, Mail, Teléfono y Tipo de Usuario.

Eliminar: borra los usuarios del sistema, manteniendo los registros históricos de auditoria.

Desbloquear: esta función permite reintentar al inicio de sesion a aquellos usuarios que han superadado la cantidad de intentos fallidos.

Cambiar Contraseña: permite ingresar una nueva contraseña para el usuario seleccionado.

Nota: no se podrán eliminar usuarios que tengan notificaciones asignadas.

Medios

Este módulo administra los medios de comunicacion a través de los cuales se notificaran las alertas del sistema.

Crear Medio: para el tipo de medio "Mail" se debera completar el Nomnbre, Destino y Dominio, en el caso del "SMS" se agrega tambien la Clave.

Editar: permite modificar la informacion del Destino, Dominio o Clave en caso de que corresponda.

Eliminar: borra los usuarios del sistema, manteniendo los registros históricos de auditoria.

Nota: no se podrán eliminar un medio de comunicacion que este asignado a una o más reglas.

Reglas

Este modulo administra las reglas que seran utilizadas por el motor de reglas para procesar las alertas del sistema .

Crear Regla: p se debera completar......

Editar: permite modificar la informacion del orden, condicion, segundos.

Eliminar: borra la regla del sistema, manteniendo los registros históricos de auditoria.

Descartar

Si una alerta cumple con la condición ingresada en la regla, la misma sera descartada y no se generara ninguna notificación

Nota: se genera un registro de auditoria indicando que la alerta fue descartada.

Consolidar

Si durante el lapso de tiempo configurado en la regla, llegan una o mas alertas, el sistema espera que expire dicho tiempo y envia una unica notificación

Esperar

Si llega una alerta que cumple con el evento ingresado o expira el tiempo configurado en la regla, se envia una unica notificacion
